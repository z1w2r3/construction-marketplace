# 建筑施工项目进度分析

分析项目当前进度,对比计划与实际,识别延期风险并提出建议措施。

你现在要执行**项目进度分析**任务。

## 执行步骤

### 1. 读取项目配置

从 `.claude/CLAUDE-construction.md` 获取:
- 项目名称
- 原文档目录路径
- 项目类型

使用 Read 工具读取配置文件。

### 2. 识别进度相关文档

使用 Bash 工具查找包含进度信息的文档:

```bash
# 查找计划相关文档
find "原文档目录" -type f \( -name "*计划*.docx" -o -name "*计划*.xlsx" -o -name "*组织设计*.docx" \)

# 查找进度报告
find "原文档目录" -type f \( -name "*月报*.docx" -o -name "*月报*.xlsx" -o -name "*周报*" \)

# 查找验收记录(反映实际进度)
find "原文档目录" -type f \( -name "*验收*.docx" -o -name "*验收*.xlsx" \)

# 查找会议纪要
find "原文档目录" -type f \( -name "*会议*.docx" -o -name "*纪要*.docx" \)
```

### 3. 验证和解析文档 ⭐ (核心改进)

#### 3.1 验证文档可读性

对找到的每个关键文档:

```
使用 mcp__construction_doc_processor__validate_document 工具:
- file_path: [文档路径]

如果验证失败:
- 记录问题
- 跳过该文档
- 继续处理其他文档
```

#### 3.2 解析文档提取进度信息

根据文档类型选择合适的工具:

```
计划类文档 (.docx, .pdf):
使用 mcp__construction_doc_processor__parse_word_document 或 mcp__construction_doc_processor__parse_pdf_document:
  - file_path: [文档路径]
  - include_tables: true

提取关键信息:
- 总工期
- 开工日期、竣工日期
- 关键节点时间
- 分部分项工程计划时间

进度报告 (.xlsx, .docx):
使用 mcp__construction_doc_processor__parse_excel_document 或 mcp__construction_doc_processor__parse_word_document:
  - file_path: [文档路径]

提取关键信息:
- 本期完成工作
- 累计完成工作
- 完成百分比
- 进度评价
```

#### 3.3 智能关键词提取 ⭐

重点提取包含以下关键词的内容:

**进度关键词库**:
```
- 计划类: "计划完成"、"计划工期"、"计划时间"、"预计"
- 实际类: "实际完成"、"已完成"、"完成率"、"累计"
- 时间类: "开工"、"竣工"、"节点"、"里程碑"
- 状态类: "延期"、"提前"、"正常"、"滞后"
- 评价类: "进度评价"、"进度分析"、"存在问题"
```

**提取策略**:
```
从解析结果中:
1. 筛选包含关键词的段落和表格
2. 提取时间信息(日期、工期)
3. 提取百分比数据(完成率)
4. 记录问题描述
5. 标注信息来源
```

### 4. 进度信息整理

#### 4.1 进度基准(计划进度)

从施工组织设计、总进度计划中提取:

```
计划信息:
- 计划总工期: X 天(Y年Z月W日 - Y'年Z'月W'日)
- 当前应完成: [阶段/工作内容]
- 关键节点计划:
  - 地基基础完成: YYYY-MM-DD
  - 主体结构完成: YYYY-MM-DD
  - 装饰装修完成: YYYY-MM-DD
  - 竣工验收: YYYY-MM-DD

来源: 《施工组织设计.docx》(原文档目录/技术资料/)
```

#### 4.2 实际进度

从月报、周报、验收记录中提取:

```
实际情况:
- 实际开工日期: YYYY-MM-DD
- 已完成工作: [列表]
- 完成百分比: X%
- 关键节点实际:
  - 地基基础完成: YYYY-MM-DD (实际)
  - 主体结构进行中: 完成Y%
  - 装饰装修: 未开始

来源: 《施工月报-2024年9月.xlsx》(原文档目录/月报/)
```

### 5. 进度对比分析

#### 5.1 计算进度偏差

```
进度对比:
- 时间进度偏差: [±X天]
  - 计划工期: Y天
  - 实际用时: Z天
  - 偏差: Z-Y = ±X天

- 工程量进度偏差: [±X%]
  - 计划完成: Y%
  - 实际完成: Z%
  - 偏差: Z-Y = ±X%

关键节点对比:
| 节点名称 | 计划时间 | 实际时间 | 偏差(天) | 状态 |
|---------|---------|---------|---------|------|
| 地基基础完成 | 2024-05-01 | 2024-05-05 | +4 | ⚠️ 延期 |
| 主体结构完成 | 2024-09-01 | 2024-09-15 | +14 | 🔴 严重延期 |
| 装饰装修完成 | 2024-12-01 | 进行中 | - | 🟡 风险 |
```

#### 5.2 识别延期部分

```
延期工作项:
1. 主体结构工程
   - 计划完成: 2024-09-01
   - 实际完成: 2024-09-15
   - 延期: 14天
   - 影响: 影响后续装饰装修开工

2. 钢筋工程
   - 计划完成: XX
   - 实际进度: 延期中
   - 原因: [提取原因]
```

### 6. 延期原因分析

从文档中提取延期原因,分类整理:

#### 6.1 客观因素

```
外部因素:
- 天气影响: 连续降雨X天
- 疫情影响: 停工Y天
- 材料供应: 钢材到货延迟Z天

来源: 《施工月报-9月》"存在问题"章节
```

#### 6.2 管理因素

```
内部因素:
- 资源配置: 人员/设备不足
- 技术问题: 技术方案调整
- 协调问题: 各方协调不畅
- 其他: [具体描述]

来源: 《工地例会纪要》
```

### 7. 风险评估 ⭐

根据偏差程度和影响,评估延期风险:

#### 🟢 低风险

```
条件:
- 进度正常或略有超前(偏差 ≤ 5%)
- 无关键节点延期
- 后续工作无阻碍

当前状态: [√/×]
```

#### 🟡 中风险

```
条件:
- 轻微延期(偏差 5-15%)
- 部分节点延期但可追赶
- 已有应对措施

当前状态: [√/×]

建议措施:
- 加强资源投入
- 优化施工组织
- 加强协调沟通
```

#### 🔴 高风险

```
条件:
- 严重延期(偏差 > 15%)
- 关键节点大幅延期
- 影响总工期

当前状态: [√/×]

紧急措施:
- 制定赶工方案
- 增加资源投入
- 优化施工工序
- 必要时调整计划
```

### 8. 建议措施

根据风险等级提供针对性建议:

#### 8.1 短期措施(1-2周)

```
立即行动:
1. 增加施工人员/设备
2. 延长每日工作时间
3. 优化施工工序
4. 加强现场协调

预期效果: 追赶进度X天
```

#### 8.2 中期措施(1-2月)

```
持续改进:
1. 优化施工组织设计
2. 改进材料采购流程
3. 加强分包管理
4. 完善进度管控机制

预期效果: 保证后续进度
```

#### 8.3 赶工方案(如严重延期)

```
赶工方案:
- 增加作业面
- 采用先进工艺
- 实施交叉作业
- 提前准备后续工作

成本影响: 增加X万元
工期影响: 可追赶Y天
```

### 9. 容错处理

